# Volt 제출 문서

## 1. 프로젝트 개요

- 프로젝트명: `Volt`
- 주제: 작사가를 위한 스마트 메모장 및 공개 영감 피드
- 기술 스택: `Next.js 16 App Router`, `TypeScript`, `Prisma`, `SQLite`
- 개발 목표:
  - 공개 피드 탐색
  - 개인 아카이브 관리
  - 가사 작성/수정/삭제
  - 라임 메모 관리
  - ko/en 다국어 지원
  - 자체 로그인 및 보호 라우트 구현

## 2. 인공지능과 함께한 설계

이 프로젝트는 AI 코딩 에이전트를 활용해 다음 설계 원칙을 기준으로 진행했습니다.

### 2-1. 화면 구조 설계

- `/explore`: 공개 가사 탐색 화면
- `/explore/[id]`: 공개 게시물 상세 화면
- `/dashboard`: 개인 아카이브 및 필터링 화면
- `/lyrics/new`: 새 글귀 작성 화면
- `/lyrics/[id]`: 가사 상세 편집 화면
- `/login`, `/signup`: 인증 화면

### 2-2. 데이터 구조 설계

Prisma 스키마 기준 핵심 모델:

- `User`
  - 이메일, 비밀번호 해시, 닉네임
- `Lyric`
  - 제목, 본문, 무드 이미지 URL, BPM, Key, 공개 여부, 폴더, 진척도, 송폼
- `RhymeNote`
  - 연결 가사 ID, 모음 패턴, 메모

### 2-3. UX 설계 방향

- 카드/패널/폼 간격을 공통 spacing scale로 통일
- 새 글귀 작성 화면을 별도 라우트로 분리
- 저장/삭제 토스트 제공
- 삭제 전 확인창 제공
- 저장되지 않은 편집 상태에서 이탈 시 경고 제공
- `Cmd/Ctrl + S` 저장 단축키 제공

## 3. 구현 내용

### 3-1. 인증 및 접근 제어

- 이메일/비밀번호 기반 자체 회원가입 및 로그인 구현
- 비밀번호는 해시 저장
- `middleware.ts`에서 `/dashboard`, `/lyrics/[id]` 보호

### 3-2. 데이터 쓰기 기능

Server Action 기반으로 아래 기능 구현:

- 가사 생성
- 가사 수정
- 가사 삭제
- 라임 메모 저장
- 라임 메모 삭제

### 3-3. 다국어 처리

- `dictionaries/ko.json`, `dictionaries/en.json` 사용
- 공통 UI 문구 및 상태 문구 분리
- 폴더/진척도/송폼 라벨도 locale별로 표시되도록 보완

### 3-4. SEO 및 메타데이터

- `generateMetadata()` 적용
- `app/sitemap.ts`, `app/robots.ts`, `app/favicon.ico` 구성

## 4. 테스트 및 검증

최종 제출 전 아래 항목을 반복 확인했습니다.

```bash

npm run build
```

### 4-2. DB 검증


### 4-3. 기능 검증 항목

- 로그인/회원가입 동작
- 비로그인 상태에서 보호 페이지 접근 차단
- 글귀 생성/수정/삭제 동작
- 라임 메모 생성/수정/삭제 동작
- 공개 피드 상세 이동
- ko/en UI 전환
- 저장/삭제 토스트 노출
- 저장 전 이탈 경고 노출
- 단축키 저장 동작
